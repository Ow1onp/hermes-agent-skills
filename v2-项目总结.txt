# Hermes-Agent-Skills v2 项目总结

## 项目概述

Hermes-Agent-Skills 最初是一个基于 Skill 的多 Agent 协作系统。

用户需要主动选择角色，例如：

* CTO
* CLI Engineer
* Documentation Engineer
* Release Publisher
* Community Manager

系统能够完成复杂任务，但随着 Skill 数量增长，出现了明显问题：

* 用户不知道应该选择哪个角色
* 用户需要频繁切换 Agent
* 新用户不了解 Constraint Engineering
* 学习成本逐渐升高

因此项目开始从：

```text
Skill First
```

转向：

```text
Task First
```

---

# Hermes v1

架构：

```text
User
 ↓
Select Skill
 ↓
Execute
```

特点：

* 强角色驱动
* 强 Prompt 驱动
* 高自由度
* 高学习成本

适合：

* 高级用户
* 熟悉 Constraint Engineering 用户

问题：

* 技能发现困难
* Agent 切换频繁
* 新用户体验差

---

# Hermes v2

核心目标：

让用户不需要了解：

* Skill
* Role
* Constraint Engineering

只需要表达目标。

例如：

```text
帮我发布项目
```

系统自动完成：

```text
Project Manager
 ↓
Release Manager
 ↓
Launch Commander
```

---

```text
Natural Language Interface
        ↓

        ↓

        ↓

        ↓

        ↓

```

# 核心组件

## Intent Router

负责：

```text
用户输入
 ↓
Task ID
```

示例：

```text
帮我发布 v2.0.0
```

↓

```json
{
  "task": "release_version"
}
```

---

## Entity Extractor

负责保留用户关键上下文。

示例：

```text
创建一个 FastAPI 项目
```

提取：

```json
{
  "technology": "FastAPI",
  "language": "Python",
  "project_type": "API"
}
```

解决：

```text
Entity Drop
```

问题。

---

## Task Registry

定义：

```yaml
create_project
fix_bug
write_docs
review_code
release_version
publish_project
```

每个任务包含：

* Intent
* Workflow
* Required Skills
* Success Criteria

---

## Task Orchestrator

自动决定：

```text
Task
 ↓
Workflow
 ↓
Skills
```

无需用户指定角色。

---

## Constraint Engine

自动生成约束工程 Prompt。

用户：

```text
帮我写 README
```

系统：

自动生成：

* Scope
* Constraints
* Success Criteria
* Deliverables

---

## Result Evaluator

负责检查：

* Completeness
* Confidence
* Entity Preservation
* Safety
* Beginner Guard
